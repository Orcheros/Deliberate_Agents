# Press Release Template

## Structure

### Headline
Action verb + company name + key benefit or news. Under 80 characters.

**Good:** "Acme Launches AI-Powered Invoice Processing, Cutting AP Time by 80%"
**Bad:** "Acme Inc. Is Pleased to Announce the Release of Their New Product"

### Subheadline
One sentence expanding on the headline. Adds context the headline couldn't fit.

### Dateline + Lead Paragraph
{CITY, STATE — Month DD, YYYY} — First paragraph answers: Who? What? When? Where? Why?

This paragraph should stand alone as a complete summary. If a journalist only reads this paragraph, they should understand the news.

### Body (3-4 paragraphs)

**Paragraph 2: The problem**
What challenge exists in the market? Why does this matter now? Use data if available.

**Paragraph 3: The solution**
What the product/feature/partnership does. Be specific about capabilities without jargon. One concrete example of how it works.

**Paragraph 4: Proof and traction**
Customer results, adoption metrics, partnerships, or expert validation. Quantify wherever possible.

### Founder/CEO Quote
2-3 sentences in first person. Should add perspective, vision, or motivation — not restate the news. Make it sound like something a human would actually say.

**Good:** "We built this because every AP team we talked to was drowning in manual invoice matching. The technology finally caught up to the problem."
**Bad:** "We are thrilled to announce this innovative solution that leverages cutting-edge AI technology."

### Customer/Partner Quote (optional)
If a customer or partner is willing to be quoted, include their perspective. Must be approved by the quoted individual. `[HUMAN GATE]`

### Boilerplate ("About {Company}")
Standard company description, 2-3 sentences. Include:
- What the company does (one sentence)
- Key metric or proof point (one sentence)
- Where to learn more (URL)

### Media Contact
```
Media Contact:
{Name}
{Title}
{Email}
{Phone}
```

## Common Mistakes

| Mistake | Fix |
|---------|-----|
| Leading with the company instead of the news | Start with what happened, not who you are |
| Jargon-heavy ("leveraging synergies") | Use plain language a journalist would use |
| No quantified impact | Add at least one number (%, $, users, time saved) |
| Quote sounds corporate | Read it aloud — would a real person say this? |
| Burying the news below background | First paragraph = the news. Everything else supports it. |
| Missing boilerplate | Always include — journalists copy-paste it |
| No media contact | Always include — this is how they reach you |
